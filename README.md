首先大模型是什么，一句话总结，大模型的本质就是个知识库的有损压缩

大家有在家里用ollama拉过大模型吗？有没看过7B参数的大模型占几个GB？7B参数的大模型按16位浮点数来算的话要14GB，那为什么7B参数只需要4GB就可以呢？

这里可以用一个算术题例子：
请告诉1 加到100 的结果是什么？我们都很清楚知道结果是5050，
那1 加到1000 呢？ 1加到10000呢？最蠢的办法就是把所有可能的结果都记住，但这不现实，再大的存储空间也不够，那人类是怎么记？答案是公式，我们小学就知道1加到100，是(1+100) * 100 / 2。那我们想想这个公式是怎么来，是不是用客观规律推导出来，记一条公式是不是比记所有答案要省心？其实大模型记得也是大量的记这种公式。当人类给了它足够多的训练数据之后，当规模达到一定的阈值，大模型自己就能总结出属于它自己的“客观规律”，4GB的公式那是非常惊人的知识量，所以就能顿悟出智能。当然人类也无法得知这些“公式”是什么，甚至也无法还原出其中的“公式”出来，这算是大模型里的黑盒子部分。当然这些公式也见得是正确，大模型只是通过概率最高的方法来拟合出“公式”。

所以这就说到大模型如何拟合出所谓的“公式”，答案是将token向量化，训练模型推导下一个token。
首先，关键字：向量化。什么意思？其实并不深奥，我用人类的例子来说明，小时候上学的时候是不是经常会听到学生说：老师，你讲得太快了，我理解不了。又或者说：你讲得太深奥了，我慢慢消化一下你说的。注意，这里说的“消化一下”，本质上就是人类将知识向量化的过程。让知识（也就是每句话里的token）转变成自己能理解的模样，这个过程本质就是向量化。那计算机怎么理解世间万物，答案是转化成数字，因为计算机只认0和1。比如说人类跟计算机说猫、狗、老虎，计算机就给3个动物贴个标签，猫0，狗1，老虎2等等，组成一个动物数组。但是这样有一个问题，如果我让你标记一个宠物数组，举例子猫0，狗1，鸡2，这个时候你发现猫和狗同时出现在动物数组和宠物数组，那显然单纯从任一一个数组都无法描述猫和狗的全部信息，这时候怎么办呢？答案是多维数组，即数组嵌套数组，[[0,0] ,[1,1] , [1,2]]，假如把他称为宠物动物二维数组，而人类对于猫、
狗、包括众多的token向量化之后可能会上百个维度，甚至上千，才能涵盖人类所理解的各种语义环境。所以向量化后的多维，本质上就是多维数组标签。有了这些标签之后，就相当于有了坐标，有了坐标就可以计算每个token的之间的距离。从而让人类感觉计算机有了理解世间万物的本领。
当人类给了一堆token的提示词之后，大模型就会跟你玩一个游戏“成语接龙”，向量化你的提示词，然后根据坐标（多维数组）去它的脑海里寻找一堆距离相近的token来反馈给你，仅此而已。

好既然我们都知道大模型是在做下一个token的高概率预测，那么就会出现一个经典头疼问题：一本正经胡说八道，专业名词叫幻觉。依然是用一个生活假设来解释一下：小明的老婆有一天问AI，你觉得我和小明他妈掉河里了，小明会先救谁？对于这道经典的送命题，大模型就会想，在我的知识库里，问这个问题的大部分是女人，所以推断小明是用户的老公，但也有可能用户是小明的小三，于是万一大模型选错了，把用户当小明的小三的角度来预测下一个token，用户可能立马去拔AI电源了，什么乱七八糟的。最要命的是它可能会非常自信的捏造事实，大模型为了把这个谎圆下去，它会一本正经地告诉你：‘根据《2026年最新水上救援法》第三条规定，应该优先救小三，因为根据浮力学原理，她穿的衣服少，下沉速度慢。——你看，法律是编的，物理学原理也是它瞎拼凑的，而且语气极其自信，这就叫典型的、无可救药的幻觉！尤其是参数越小的大模型，幻觉就越严重。那是不是小参数的大模型就没有用武之地了？有的，包有的。你想想很多物理硬件可能并不具备很强大的GPU硬件条件，那这些硬件需要用低延迟的AI怎么办？只能是部署小参数的大模型。那业界怎么解决这个头疼的幻觉问题呢？答案就是大名鼎鼎的RAG技术。

我们想想小时候我们上学那会都做过试卷，试卷里通常简单的放前面，难的硬骨头都放后面，你想下前面大部分的是什么题？选择题和判断题对不对？后面的是什么题？问答题对不对？大模型参数小意味着不够聪明，那就别让它做问答题，尽量让它做选择题。那怎样将用户的问答题转化成选择题来答呢？回到我们刚才举的例子，小明的老婆问AI：我和小明他妈掉河里了，小明会先救谁？当这个问题发送给大模型之前会经过一个后端服务（网关），后端服务会先将这个提示词向量化，然后去向量数据库里匹配相近数据，这个时候假如搜到了一条文档：{description: 小明的性格特征, value:小明是妻管严}，这是个至关重要的线索，打破送命题的关键。后端就会整合提示词：我和小明他妈掉河里了，小明会先救谁？PS：小明是妻管严。然后发给大模型。你看都提示到这份上了，总不会答错了吧，然后大模型就会回答：当然是救你啦，小明本来就是个疼老婆的人。然后用户突然觉得几百个B的参数大模型还没自家的7个B的好用。其实这种通过搜索相关文档再整合提示词的技术，就叫RAG技术（检索增强生成），所以RAG的本质就是提示词工程。这样理解是不是就很清晰了？

好，既然我们弄明白了RAG技术的原理，我们当然不能止步于此，毕竟RAG大多数时候其实也就是让答案更符合人类的预期而已，而人类可能要AI做得更多，人类希望AI除了嘴巴甜以外，还要长眼睛会看，长手会做事。这就诞生了另一门技术AI Agent。那大伙有没有一个问题，究竟大模型是怎样看我所看，做我所做的呢？

首先，我们先来破解第一个错觉：大模型其实根本没有眼睛，也没有手，它自始至终，依然只是那个只会算概率的‘文字接龙机器’。
那他是怎么看得见呢？其实还是回到最开始我们探讨的那个问题。大模型如何理解世间万物：向量化！简单来讲就是先把图片像切豆腐一样切开，然后每一块都向量化就会有专属的多维数组，通过匹配多个多维数组的组合，就能识别到这个图是狗还是猫还是世间万物。仍然还是那句话，向量化就是大模型在理解这个世界的最重要途径。
那openclaw是的“眼睛”是怎么运作的？比如你叫openclaw帮我打开浏览器，openclaw就会立即对你屏幕疯狂截图，有了图片就好办了，交给大模型去理解，得出浏览器在桌面的像素坐标，接下来就将鼠标移过去双击就完成了。这就是为什么这玩意耗token。特别是图片理解。这么说，平时让AI大模型回答一条文本问题，可能也就4、5千token可能就顶填了，但是如果你让它去分析一张图片，有可能很轻松就上万，而且为了确保鼠标能正确移到浏览器那，open claw不会只截取一张图，而是多张。反正有人做过一个测试，让openclaw去linux系统里设置一个定时任务，5毛没了。我做的应用，让大模型分析一张化验单图片里的指标（1-3毛）。这就是人类懒的代价，你不想多动手，就只能让口袋里的钱包动起来，反正总有一个要动。
